Q4 Lesson 2: Script Placement and Basic Coding Guidelines

Broswer Output:

    


What Was Learned

I learned that the placement of JavaScript significantly affects when and how code is executed within an HTML document. 
Understanding these syntax guidelines is crucial for preventing script errors. Key takeaways include:

HEAD Section: Scripts placed here are pre-loaded and executed before the rest of the page, making it ideal for function definitions.

BODY Section: Scripts here execute as the page loads, useful for displaying immediate content via document.write().

External JS (.js): Using an external file via the src attribute allows for cleaner code and reusability across multiple webpages.

Case Sensitivity: Variable names are unique; for example, myHome is not the same as MyHome.

Syntax Guidelines: JS ignores extra white space between code lines, and the backslash (\) can be used to break up long lines of text without breaking the command.

Commenting: Use // for single-line notes and /* ... */ for multiple-line or block comments.
